    • 开始使用 - 入门指南

    byui基于gulp构建的前端框架，是js模块化开发解决方案。在layui的基础上进行了重新封装，整个框架只引用了jquery第三方库，目前我们PC端、手机端项目前端都是基于jquery构建，项目中前端引用框架、插件过多，并且并不是全部框架功能都有使用，严重影响前端页面加载速度，鉴于以上原因，把常用的功能基于layui重新封装了一下。会重新封装PC、移动端两个版本，希望对以后项目有所帮助。
    兼容性和面向场景
    byui 不兼容IE6/7。
    byui目录结构 
    byui.code
    1.   ├─css //css目录
    2.   │  │─modules //模块css目录（一般如果模块相对较大，我们会单独提取，比如下面三个：）
    3.   │  │  ├─laydate
    4.   │  │  ├─layer
    5.   │  │  └─layim
    6.   │  └─byui.css //核心样式文件
    7.   ├─font  //字体图标目录
    8.   ├─images //图片资源目录（目前只有layim和编辑器用到的GIF表情）
    9.   │─lay //模块核心目录
    10.   │  └─modules //各模块组件
    11.   │─byui.js //基础核心库
    12.   └─byui.all.js //包含byui.js和所有模块的合并文件
    快速上手 
    模块化引用
    1. ./byui/css/byui.css
    2. ./byui/byui.js //提示：如果是采用非模块化方式（最下面有讲解），此处可换成：./byui/byui.all.js
    2. <script src="../byui/byui.js"></script>
    3. <script>
    4. //一般直接写在一个js文件中
    5. byui.use(['layer', 'form'], function(){
    6.   var layer = byui.layer
    7.   ,form = byui.form;
    8.   
    9.   layer.msg('Hello World');
    10. });
    11. </script>

    如果你想采用非模块化方式（即所有模块一次性加载），你也可以按照下面的方式使用：

    1. <script src="../byui/byui.all.js"></script>
    2. <script>
    3. //由于模块都一次性加载，因此不用执行 byui.use() 来加载对应模块，直接使用即可：
    4. ;!function(){
    5.   var layer = byui.layer
    6.   ,form = byui.form;
    7.   
    8.   layer.msg('Hello World');
    9. }();
    10. </script>
    模块化的用法（推荐）

    我们推荐你遵循 byui 的模块规范建立一个入口文件，并通过 byui.use() 方式来加载该入口文件，如下所示：

    1. <script>
    2. byui.config({
    3.   base: '/res/js/modules/' //你存放新模块的目录，注意，不是layui的模块目录
    4. }).use('index'); //加载入口
    5. </script>    
    6.
    上述的 index 即为你 /res/js/modules/ 目录下的 index.js，它的内容应该如下： 
    1. /**
    2.   项目JS主入口
    3.   以依赖layui的layer和form模块为例
    4. **/    
    5. byui.define(['layer', 'form'], function(exports){
    6.   var layer = byui.layer
    7.   ,form = byui.form;
    8.   
    9.   layer.msg('Hello World');
    10.   
    11.   exports('index', {}); //注意，这里是模块输出的核心，模块名必须和use时的模块名一致
    12. });    
    13.
