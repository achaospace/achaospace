    底层方法

    定义模块

    方法：byui.define([mods], callback)

    通过该方法可定义一个 Bayui模块。参数mods是可选的，用于声明该模块所依赖的模块。callback即为模块加载完毕的回调函数，它返回一个exports参数，用于输出该模块的接口。

    1. byui.define(function(exports){
    2.   //do something
    3.   
    4.   exports('demo', function(){
    5.     alert('Hello World!');
    6.   });
    7. });
    8.

    exports是一个函数，它接受两个参数，第一个参数为模块名，第二个参数为模块接口，当你声明了上述的一个模块后，你就可以在外部使用了，demo就会注册到bayui对象下，即可通过 byui.demo() 去执行该模块的接口。

    你也可以在定义一个模块的时候，声明该模块所需的依赖，如：

    1. byui.define(['layer', 'laypage'], function(exports){
    2.   //do something
    3.   
    4.   exports('demo', function(){
    5.     alert('Hello World!');
    6.   });
    7. });
    8.

    上述的['layer', 'laypage']即为本模块所依赖的模块，它并非只能是一个数组，你也可以直接传一个字符型的模块名，但是这样只能依赖一个模块。

    加载所需模块

    方法：byui.use([mods], callback)

    Byui的内置模块并非默认就加载的，他必须在你执行该方法后才会加载。它的参数跟上述的 define方法完全一样。
    另外请注意，mods里面必须是一个合法的模块名，不能包含目录。 

    1. byui.use(['laypage', 'layedit'], function(){
    2.   var laypage = byui.laypage
    3.   ,layedit = byui.layedit;
    4.   
    5.   //do something
    6. });
    7.

    该方法的函数其实返回了所加载的模块接口，所以你其实也可以不通过byui对象赋值获得接口

    1. byui.use(['laypage', 'layedit'], function(laypage, layedit){
    2.   
    3.   //使用分页
    4.   laypage();
    5.   
    6.   //建立编辑器
    7.   layedit.build();
    8. });
    9.

    动态加载CSS

    方法：byui.link(href)

    href即为css路径。它一般只是用于动态加载你的外部CSS文件。

    本地存储

    本地存储是对 localStorage 和 sessionStorage 的友好封装，可更方便地管理本地数据。

    此处后续添加对cookie的封装（未完成）

    • localStorage 持久化存储：byui.data(table, settings)，数据会永久存在，除非物理删除。
    • sessionStorage 会话性存储：byui.sessionData(table, settings)，页面关闭后即失效。

    上述两个方法的使用方式是完全一样的。其中参数 table 为表名，settings是一个对象，用于设置key、value。下面与 byui.data 方法为例： 

    1. //【增】：向test表插入一个nickname字段，如果该表不存在，则自动建立。
    2. byui.data('test', {
    3.   key: 'nickname'
    4.   ,value: 'XXX'
    5. });
    6.  
    7. //【删】：删除test表的nickname字段
    8. byui.data('test', {
    9.   key: 'nickname'
    10.   ,remove: true
    11. });
    12. byui.data('test', null); //删除test表
    13.   
    14. //【改】：同【增】，会覆盖已经存储的数据
    15.   
    16. //【查】：向test表读取全部的数据
    17. var localTest = byui.data('test');
    18. console.log(localTest.nickname); //获得“XXX”
    19.
    获取设备信息

    方法：byui.device(key)，参数key是可选的

    此处后续移动版添加对手机型号的判断(未完成)

    1. var device = byui.device();
    2. //device即可根据不同的设备返回下述不同的信息
      1. {
      2.   os: "windows" //底层操作系统，windows、linux、mac等

      4.   ,weixin: false //是否微信环境
      5.   ,android: false //是否安卓系统
      6.   ,ios: false //是否ios系统
      7. }

    其它

    其实除此之外，byui.js内部还提供了许多底层引擎，他们同样是整个Byui框架体系的有力支撑，所以有必要露个脸，尽管你可能并不会用到：

    方法/属性 描述
    byui.cache 静态属性。获得一些配置及临时的缓存信息
    byui.extend(options) 拓展一个模块别名，如：byui.extend({test: '/res/js/test'})
    byui.each(obj, fn) 对象（Array、Object、DOM对象等）遍历，可用于取代for语句
    byui.getStyle(node, name) 获得一个原始DOM节点的style属性值，如：byui.getStyle(document.body, 'font-size')
    byui.img(url, callback, error) 图片预加载
    byui.sort(obj, key, desc) 将数组中的对象按某个成员重新对该数组排序，如：byui.sort([{a: 3},{a: 1},{a: 5}], 'a')
    byui.router() 获得location.hash路由，目前在Byui中没发挥作用。对做单页应用会派上用场。
    byui.hint() 向控制台打印一些异常信息，目前只返回了error方法：byui.hint().error('出错啦')
    byui.stope(e) 阻止事件冒泡
    byui.onevent(modName, events, callback) 自定义模块事件，属于比较高级的应用。有兴趣的同学可以阅读byui.js源码以及form模块
    byui.event(modName, events, params) 执行自定义模块事件，搭配onevent使用
    byui.factory(modName) 用于获取模块对应的 define 回调函数
    第三方支撑

    Byui部分模块依赖jQuery（比如layer），但是你并不用去额外加载jQuery。Byui已经将jQuery最稳定的一个版本改为Byui的内部模块，当你去使用 layer 之类的模块时，它会首先判断你的页面是否已经引入了jQuery，如果没有，则加载内部的jQuery模块，如果有，则不会加载。
